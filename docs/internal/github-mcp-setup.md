# GitHub MCP Setup Prompt for Claude Code

Paste this into Claude Code to set up the GitHub MCP connector.

---

```
Set up the GitHub MCP server for this project so I can commit, push, and 
manage the repo directly from Code sessions without manual git commands.

Do the following:

1. Check if a GitHub MCP server is already configured:
   cat ~/.claude/mcp.json 2>/dev/null || echo "No mcp.json found"
   cat ~/.claude/settings.json | grep -A5 "github" 2>/dev/null || echo "No github config"

2. If not configured, add the official Anthropic GitHub MCP server.
   The standard config for settings.json is:

   In the "mcpServers" block, add:
   {
     "github": {
       "type": "url",
       "url": "https://github.mcp.claude.com/mcp",
       "autoApprove": [
         "list_repos",
         "get_file_contents", 
         "create_or_update_file",
         "push_files",
         "list_commits",
         "get_commit",
         "create_branch",
         "list_branches",
         "search_code"
       ]
     }
   }

3. After adding, restart Claude Code and verify the GitHub MCP tools
   are available by running:
   /mcp

4. Test the connection by listing the FFP repo contents:
   List the files in carterspot/financial-freedom-platform on the main branch.

5. If authentication is required, follow the OAuth flow that appears.

Report what was found in step 1, what was changed in step 2, and whether 
the test in step 4 succeeded. Do not modify any project files during setup.
```

---

## What GitHub MCP unlocks for FFP workflow

Once active, Code can do all of this in a single session without manual git:

| Task | Before MCP | After MCP |
|------|-----------|-----------|
| Commit a build | You manually git add/commit/push | Code commits directly |
| Read current module | You upload the .jsx file | Code reads from repo |
| Update docs/index.html | You edit on GitHub.com | Code edits and pushes |
| Check last commit | You run git log | Code queries commit history |
| Create feature branch | Manual | Code creates branch, works on it |
| Read CLAUDE.md | Code reads automatically | Same — already works |

## Suggested workflow after GitHub MCP is active

When starting a new build session, Code can:
1. Read the current `modules/{name}.jsx` directly from GitHub
2. Read `docs/project-instruction.md` for full context
3. Make changes, run build, audit
4. Commit with a descriptive message
5. Update docs in the same commit

No more file uploads. No more manual git. Session starts clean every time.

## Important: What GitHub MCP does NOT solve

- Artifact rendering still requires pasting JSX into the dedicated Claude.ai chat
- Artifact URLs are still generated by Claude.ai — not controllable via API
- The CTO chat (this chat) still writes the prompts and specs for Code to execute
- GitHub MCP is for Code sessions only — not available in this claude.ai chat

## Auto-approve note

The `autoApprove` list above covers read + write operations for FFP.
If Code asks for permission on a GitHub tool not in the list, select
"Yes, allow for this project" to add it to the project-level whitelist.
