## LoanTracker v1.0 — Bug Report

### 🔴 Critical

**1. Profile ID doesn't use PIN-based stable format**
Line 221: New profiles get `id: generateId()` — a random ID. The spec requires `id: "pin_" + pin.toLowerCase().replace(/\s+/g, "_")`. Cross-module consequences: if a user creates a profile in LoanTracker, then opens CardTracker and hits "Recover by PIN," the stable `pin_smithfamily` key won't match the random ID stored in `cc_profiles`. Recovery breaks.

---

### 🟡 Medium

**2. Charts tab applies lump sums to every individual loan chart**
Line 1166: `validLoans.map(l => <BalanceChart schedule={amortizeLoan(l, extraBudget, lumpSums)} .../>)` — a $5,000 lump sum is independently subtracted from *each* loan's balance chart. A user with 3 loans sees each one drop by $5k. In reality a lump sum is applied once to the priority loan.

**3. `singleLoan` holds stale loan snapshot**
Line 927: `singleSchedule = amortizeLoan(singleLoan, ...)` uses the loan object snapshotted at time of selection. If the user edits a loan (balance, rate, payment) and reopens the planner, the Single Loan tab shows old data until they click the loan button again. Fix: derive live data with `validLoans.find(l => l.id === singleLoan?.id)`.

**4. "Interest Saved" label is hardcoded to Avalanche**
Line 1020: `{l: "Interest Saved (Avalanche)", v: fmt$(Math.abs(...))}` — always says "Avalanche" even when snowball saves more interest or they're equal.

---

### 🟢 Minor / Polish

**5. Mobile backup button is hidden with no alternative path**
Line 1353: `{!isMobile && <button onClick={() => setShowBackup(true)}>📦</button>}` — users on mobile have no way to export their loans.

**6. Quick Pay doesn't auto-log to Progress tab**
`handleQuickPay` (line 1313) updates the balance but doesn't write an entry to `lt_logs_{profileId}`. Users must manually duplicate the payment in the Progress tab to maintain log accuracy.

**7. Loans excluded from planner with no explanation**
Line 920: `validLoans` silently filters out any loan missing a payment amount. Users with incomplete loans don't know why those loans don't appear in the planner, refinance tab, or what-if chat.

---

### v1.1 Scope Recommendation

The most impactful things to fix, roughly in priority order:

1. **Profile PIN ID fix** (critical, 2-line change)
2. **Stale singleLoan fix** (medium, clean it up with a derived value)
3. **Charts lump sum fix** (medium, pass `[]` for lump sums in per-loan charts)
4. **Interest Saved label** (minor, dynamic label)
5. **Mobile backup** (minor, add backup button to profile modal on mobile)
6. **Auto-log Quick Pay → Progress** (nice UX, links the two flows together)
7. **"Excluded loans" warning** in planner header
