## Objective

Add two things to RetirementModule:

1. **Spending profile validation** — warn the user if their Target Monthly Income in retirement is below their current minimum monthly expenses (from `ffp_baseline_`).
2. **Baseline context in AI** — pass `ffp_baseline_` into the AI retirement analysis so Claude can reason about whether projected withdrawals actually cover the user's cost of living.

**Scope: exactly these two changes. No new tabs, no UI redesign.**

---

## Change 1 — Read `ffp_baseline_` on Init

In the `init()` function (the `useEffect` that loads profile data), after loading the active profile, add:

```javascript
const baseline = await storeGet(`ffp_baseline_${profile.id}`, true) || null;
setBaseline(baseline);
```

Add `const [baseline, setBaseline] = useState(null);` to the component's state declarations.

Also refresh baseline on profile switch.

---

## Change 2 — Target Income Validation Warning

In the **Overview tab**, wherever `targetMonthlyIncome` is displayed or edited, add a warning indicator when:

```javascript
baseline !== null &&
baseline.amount > 0 &&
retProfile.targetMonthlyIncome > 0 &&
retProfile.targetMonthlyIncome < baseline.amount
```

**Inline warning** — amber color, appears directly below the Target Monthly Income field:

```
⚠️  Your target ($X,XXX/mo) is below your current minimum monthly
    expenses ($X,XXX/mo from SpendingTracker). Consider increasing
    your target to at least $X,XXX/mo.
```

- Use `fmt$()` for all dollar amounts
- Only show when the condition is true — no warning when target ≥ baseline or baseline is null
- Do NOT block the user from saving their target — this is advisory only

Also show this warning in the **withdrawal plan comparison cards** if the monthly income generated by the locked plan is below baseline:

```
⚠️  Below your $X,XXX/mo spending baseline
```

Small amber pill, appears under the monthly income figure on the relevant plan card.

---

## Change 3 — Baseline Context in AI Advisor

In the AI advisor tab (Plan & AI tab), wherever the retirement analysis prompt is built, add baseline context if available:

```javascript
const baselineContext = baseline
  ? `\n\nCurrent Spending Profile (from SpendingTracker):\n- Minimum monthly expenses (essential spending): ${fmt$(baseline.amount)}\n- Top essential categories: ${baseline.breakdown.slice(0,5).map(b => `${b.catName} $${b.avg.toFixed(0)}/mo`).join(", ")}\n- This represents the user's real cost of living floor. Their retirement income must exceed this to cover basic needs.`
  : "\n\nCurrent Spending Profile: Not available (user hasn't run SpendingTracker v1.8+ yet).";
```

Append `baselineContext` to the existing retirement analysis prompt. Keep all existing prompt content — this is additive only.
