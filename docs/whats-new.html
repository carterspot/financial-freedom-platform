SpendingTracker
v1.1
March 2026
🔧 Bug Fixes — v1.1

Profile switcher now works

The nav bar avatar button was not opening the profile dropdown. A new ProfileDropdown component now shows all profiles with an active checkmark. Switching profiles reloads all data — accounts, transactions, categories, and rules — for the selected profile.

JSON and CSV export now working

Both export buttons were silently failing. The download anchor was detached from the DOM before it could fire. Fixed by appending the anchor to the document body before clicking and removing it after. Downloads now work reliably across all browsers.

Flagged transactions now clearly surfaced

After import, low-confidence AI assignments now show an amber left border and a REVIEW badge on each flagged transaction row. A dismissible amber banner in the Transactions tab shows the total count of unreviewed items so nothing slips through.

Rules now apply retroactively

Creating or editing a rule now immediately re-runs against all existing unlocked transactions. A confirmation shows how many transactions were updated. Confirming a Needs Review item also auto-creates a rule and applies it retroactively — next month's import will be faster automatically.

✨ New Module — v1.0

CSV import from any bank or credit card statement

Import spending data from Chase, Bank of America, Amex, or any institution. The column mapper lets you match your CSV's columns to the required fields — and remembers your setup for every future import from that account.

Sign-flip toggle per account

Credit card statements show purchases as positive — bank statements show them as debits. A per-account flip toggle normalizes everything automatically and is remembered for every future import.

AI batch categorization on import

After rules run, all uncategorized transactions are sent to AI in a single API call. High-confidence matches are assigned silently. Low-confidence ones go to a Needs Review queue — confirm them and rules are auto-created for next time.

Rules engine — full CRUD + AI suggest

SpendingTracker owns the shared platform rules engine. Create rules manually, create them from any transaction row, or let AI suggest rules for your unruled merchants. Rules run before AI on every future import.

Actuals view with 3-month rolling average

The Summary tab shows what you actually spent per category this month alongside a 3-month rolling average. Delta is color-coded — green means under average, amber is 10–25% over, red is 25%+ over. No manual budget entry required.

Three-tab layout: Transactions / Summary / Rules

Transactions shows your full monthly list with search and category filter. Summary shows category-level actuals vs rolling average. Rules is the full rules management interface.

Deduplication on import

Before importing, the module checks for existing transactions with matching date, description, and amount. Duplicates are flagged so you can choose to skip or overwrite — safe to re-import the same statement without doubling entries.

Category seeding fallback

If IncomeTracker hasn't been run yet, SpendingTracker seeds the shared FFP category list (56 categories, 13 sections) automatically on first run. The category system is always ready regardless of module order.

IncomeTracker
v1.0 — Initial Release
March 2026
✨ New Module

Track every income stream in one place

Add any income source — W2, self-employment, rental, dividends, side business, benefits, or other. Each stream stores amount, frequency, stability rating, after-tax flag, start/end dates, category, and color.

Automatic monthly normalization

Enter your actual pay amount at whatever frequency you're paid — weekly, bi-weekly, semi-monthly, monthly, quarterly, annual, or one-time. The dashboard normalizes everything to a monthly total automatically.

Stability ratings

Mark each stream as Stable, Mostly Stable, Variable, or Irregular. The dashboard shows your overall income stability at a glance — and this data will feed directly into the AI Advisor when it launches.

Summary dashboard

Total monthly income, annual total, breakdown by income type, and stream count — all calculated live. One-time income is shown separately so it doesn't inflate your recurring total.

Seeds the shared category system

On first run, IncomeTracker seeds the platform-wide category list (56 categories across 13 sections) used by all FFP modules. Every module that launches after this — Spending, AI Advisor — reads from the same shared list.

Shared profile + API key system

Uses the same cc_profiles and cc_apikey storage as all other modules. Set up once and it works across the entire platform.

Export JSON + CSV, Import JSON

Full JSON backup and restore (replace or merge). CSV export for spreadsheet analysis. Export before making major changes.

DebtTracker
v1.1
March 2026
🔧 Bug Fixes — v1.1

Standalone profile creation screen

DebtTracker previously assumed a profile already existed from CardTracker or LoanTracker. New users now see a dedicated welcome screen to create their profile — name, avatar color, and Recovery PIN — before entering the app. Existing users are unaffected.

✨ New Module — v1.0

Unified credit card + loan debt tracker

DebtTracker merges CardTracker and LoanTracker into one artifact. Every credit card and installment loan in a single view with one unified payoff schedule.

Cross-debt avalanche and snowball engine

The payoff engine ranks all debts — cards and loans — together. Your extra dollars flow to the highest-priority debt regardless of type. True cross-debt optimization.

Five-tab planner: Schedule, Single Debt, Charts, What-If AI, Progress

Full planner suite covering every angle of debt payoff — strategy, individual amortization, visual charts, AI chat, and payment tracking.

Strategy Builder folded into What-If AI tab

The 5-question AI questionnaire lives inside the What-If tab as a collapsible "Build My Strategy" panel — keeps the tab bar clean without losing the feature.

Debt grouping toggle: by type or interleaved by APR

Default view shows cards first, then loans. Toggle to interleaved mode to see all debts ranked purely by APR — useful for understanding true payoff priority.

One-click import from CardTracker and LoanTracker

A banner on first load detects existing CardTracker or LoanTracker data and offers to import it. No re-entry required — your debts come with you.

Shared profile and API key system

Uses the same cc_profiles and cc_apikey storage as CardTracker and LoanTracker. Set up your profile once — all three modules share it automatically.

CardTracker
v3.1
March 2026
🔧 Bug Fixes

Planner values now persist across open/close

Extra/mo and lump sum amounts were resetting every time you closed and reopened the planner. Both now save automatically and reload exactly where you left them.

Apply button with "Saved ✓" confirmation

The Extra/mo field now has an explicit Apply button. After saving you'll see a "Saved ✓" confirmation so you know your change took effect.

PAID column correctly shows lump sum totals

On LUMP rows in the payoff schedule, the PAID column now shows the correct combined amount: regular payment + lump sum. Previously it was only showing the regular payment.

Individual card payment sub-line reflects lump sums

The per-card payment breakdown now correctly includes lump sum amounts in both Priority and Split Even distribution modes.

Recalculate minimums checkbox persists across sessions

The "Recalculate minimums monthly" toggle now saves its state and reloads correctly on next session.

ℹ️ tooltip uses proper InfoModal component

The tooltip on the Recalculate minimums checkbox previously used window.alert() which was blocked in some browsers. Replaced with a proper inline modal — works everywhere.
