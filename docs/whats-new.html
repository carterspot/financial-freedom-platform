DebtTracker
v1.1
March 2026
🔧 Bug Fixes — v1.1

Standalone profile creation screen

DebtTracker previously assumed a profile already existed from CardTracker or LoanTracker. New users now see a dedicated welcome screen to create their profile — name, avatar color, and Recovery PIN — before entering the app. Existing users are unaffected.

✨ New Module — v1.0

Unified credit card + loan debt tracker

DebtTracker merges CardTracker and LoanTracker into one artifact. Every credit card and installment loan in a single view with one unified payoff schedule.

Cross-debt avalanche and snowball engine

The payoff engine ranks all debts — cards and loans — together. Your extra dollars flow to the highest-priority debt regardless of type. True cross-debt optimization.

Five-tab planner: Schedule, Single Debt, Charts, What-If AI, Progress

Full planner suite covering every angle of debt payoff — strategy, individual amortization, visual charts, AI chat, and payment tracking.

Strategy Builder folded into What-If AI tab

The 5-question AI questionnaire lives inside the What-If tab as a collapsible "Build My Strategy" panel — keeps the tab bar clean without losing the feature.

Debt grouping toggle: by type or interleaved by APR

Default view shows cards first, then loans. Toggle to interleaved mode to see all debts ranked purely by APR — useful for understanding true payoff priority.

One-click import from CardTracker and LoanTracker

A banner on first load detects existing CardTracker or LoanTracker data and offers to import it. No re-entry required — your debts come with you.

Shared profile and API key system

Uses the same cc_profiles and cc_apikey storage as CardTracker and LoanTracker. Set up your profile once — all three modules share it automatically.

CardTracker
v3.1
March 2026
🔧 Bug Fixes

Planner values now persist across open/close

Extra/mo and lump sum amounts were resetting every time you closed and reopened the planner. Both now save automatically and reload exactly where you left them.

Apply button with "Saved ✓" confirmation

The Extra/mo field now has an explicit Apply button. After saving you'll see a "Saved ✓" confirmation so you know your change took effect.

PAID column correctly shows lump sum totals

On LUMP rows in the payoff schedule, the PAID column now shows the correct combined amount: regular payment + lump sum. Previously it was only showing the regular payment.

Individual card payment sub-line reflects lump sums

The per-card payment breakdown now correctly includes lump sum amounts in both Priority and Split Even distribution modes.

Recalculate minimums checkbox persists across sessions

The "Recalculate minimums monthly" toggle now saves its state and reloads correctly on next session.

ℹ️ tooltip uses proper InfoModal component

The tooltip on the Recalculate minimums checkbox previously used window.alert() which was blocked in some browsers. Replaced with a proper inline modal — works everywhere.
